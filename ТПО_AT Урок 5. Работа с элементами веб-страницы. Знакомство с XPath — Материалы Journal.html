Знакомство с XPath

XPath (XML Path Language) появился как стандарт, разработанный W3C (World Wide Web Consortium) в 1999 году для работы с XML-документами. В то время XML (Extensible Markup Language) был основным форматом для хранения и передачи данных. Он использовался везде — от конфигурационных файлов до обмена данными между системами. Но работа с большими XML-файлами требовала мощного инструмента для навигации по их структуре, чтобы находить и обрабатывать нужные данные.

XPath стал таким инструментом. Это язык запросов, который позволяет обращаться к конкретным узлам в XML-документе, оперировать значениями и извлекать данные. Благодаря своей универсальности XPath быстро стал не только стандартом для XML, но и начал использоваться в других технологиях, включая HTML-документы.

Что такое XPath?

XPath — это язык для поиска и выбора информации в документах XML и HTML. Его можно сравнить с навигацией по карте: XPath указывает точный путь к нужному элементу, даже если документ имеет сложную, вложенную структуру.

Основной задачей XPath является определение пути к элементу или группе элементов в иерархическом дереве документа. HTML-страницы, как и XML, представляют собой дерево элементов, что делает XPath незаменимым инструментом для работы с веб-документами.

Зачем нужен XPath?

XPath — это инструмент для автоматизированного взаимодействия с веб-страницами и XML-документами. Веб-приложения часто состоят из множества элементов, таких как кнопки, формы, поля ввода и ссылки. XPath позволяет находить эти элементы и взаимодействовать с ними, что делает его основой для автоматизированного тестирования и веб-скрапинга (извлечения данных с веб-страниц).

XPath работает на основе древовидной структуры документа, где каждый элемент представлен как узел (node). HTML-документ начинается с корневого узла <html> и далее делится на ветви, такие как <head> и <body>. Каждая ветвь может содержать дочерние элементы.

Пример структуры HTML:

<html>
    <body>
        <div id="main">
            <h1>Заголовок</h1>
            <p>Текст параграфа</p>
        </div>
    </body>
</html>

XPath позволяет нам обращаться к любому узлу в этой структуре. Например:

  • Путь /html/body/div/h1 указывает на заголовок <h1> внутри <div>.
  • Путь //p находит все элементы <p> в документе.

Основные возможности XPath

  1. XPath позволяет выбрать все элементы с заданным именем.
  2. Фильтры в квадратных скобках помогают сузить поиск.
  3. XPath поддерживает функции для более сложных выборок.
  4. XPath позволяет выбирать элементы по индексу.

Применение XPath в реальной жизни

  1. Работа с DevTools браузера:
    Современные браузеры, такие как Chrome, позволяют использовать XPath для поиска элементов на странице. Достаточно открыть инструменты разработчика (DevTools), перейти на вкладку «Elements» и использовать XPath в функции поиска.
  2. Инструменты автоматизации:
    В Selenium тестировщики пишут XPath-запросы, чтобы найти элементы для тестов. Например: find_element(By.XPATH, «//button[text()=’Войти’]»)
  3. Веб-скрапинг:
    В BeautifulSoup или Scrapy XPath помогает извлекать нужные данные с веб-страниц. Например: element = tree.xpath(«//h1/text()»)

Преимущества и ограничения XPath

Преимущества:

  • Позволяет точно выбирать элементы, даже в сложных документах.
  • Универсальность: работает как с XML, так и с HTML.
  • Поддерживает сложные фильтры и функции.

Ограничения:

  • Абсолютные пути зависят от структуры документа, что делает их хрупкими при изменении страницы.
  • XPath может быть менее эффективным в производительности по сравнению с другими методами (например, CSS-селекторами).
Практическое задание Вспомните что такое DevTools, для этого откройте его, возьмите поисковик inspect и выделите разные объекты на странице следя за тем как они отображаются в DOM страницы.
Сайт для тренировки можно выбрать на ваше усмотрение.

Типы выражений XPath

Один из самых важных аспектов работы с XPath — это выбор правильного типа выражения. Знание типов выражений XPath позволяет находить элементы, даже если структура документа сложна или изменяется.

XPath предоставляет два основных типа выражений: абсолютные и относительные пути. Каждый из них имеет свои преимущества и подходит для определённых задач. Рассмотрим их более подробно.

Абсолютные выражения XPath

Абсолютный путь всегда начинается с корневого узла документа (/) и представляет собой полный маршрут к нужному элементу. Это похоже на полный адрес, который указывает, как добраться до конкретного дома в большом городе.

Пример структуры HTML:

<html>
    <body>
        <div>
            <h1>Заголовок</h1>
            <p>Первый абзац</p>
            <p>Второй абзац</p>
        </div>
    </body>
</html>

Абсолютный путь к элементу <h1>:

/html/body/div/h1

Особенности абсолютных выражений:

  1. Прямой маршрут: Абсолютные выражения точно описывают путь от корневого узла до элемента.
  2. Жёсткая привязка: Если структура документа изменится, даже незначительно (например, добавится ещё один уровень вложенности), выражение перестанет работать.

Когда использовать абсолютные выражения:

  • Когда структура документа стабильна и редко меняется.
  • Когда нужно точно указать путь для одного элемента.

Недостатки абсолютных выражений:

  • Хрупкость: Любое изменение в структуре делает путь недействительным.
  • Неудобство: Для сложных и глубоко вложенных элементов пути могут быть длинными и трудно читаемыми.

Абсолютный путь проложить достаточно просто, надо:

  • Открыть вкладку elements в DevTools
  • С помощью Inspect выбрать необходимый элемент на web-страничке
  • Правой кнопкой мыши нажать по найденному элементу в elements
  • В выпадающем меню выбрать Copy, далее Copy XPath
  • Сочетанием клавиш CTRL+A или кнопкой правого меню «Вставить» вставить скопированный путь.

Относительные выражения XPath

Относительные пути начинаются с двойного слэша (//) или текущего узла (.). Они не требуют указания полного маршрута, а позволяют искать элементы относительно текущего положения в структуре документа.

Пример относительного пути:

//p

Этот путь найдёт все элементы <p> в документе, независимо от их местоположения.

Особенности относительных выражений:

  1. Гибкость: Относительные выражения работают независимо от изменений в верхних уровнях структуры.
  2. Универсальность: Их можно применять для поиска множества элементов.

Типы относительных путей:

  • Поиск на любом уровне:
    //div — находит все <div> в документе.
  • Относительно текущего узла:
    ./p — ищет элементы <p>, которые являются потомками текущего узла.
  • Комбинация:
    //div//p — находит все <p> внутри всех <div>.

Когда использовать относительные выражения:

  • Когда структура документа сложна и может изменяться.
  • Для работы с большими документами, где требуется искать элементы на разных уровнях.

Недостатки относительных выражений:

  • Может быть неоптимальным: Например, //div перебирает все узлы в документе, что может замедлить работу.
  • Менее точный поиск: Требуется дополнительная фильтрация, чтобы избежать выбора ненужных элементов.

Относительный путь является наиболее эффективным для поиска локаторов в автотестировании, но что бы научиться искать относительный путь, необходимо выучить базовый синтаксис и функции XPath.

Для того что бы проверить правильность составления относительного пути нужно:

  • Открыть вкладку elements в DevTools
  • С помощью Inspect выбрать необходимый элемент на web-страничке
  • Изучить структуру этого элемента, его дочерние теги, его родительский тег и т.д.
  • Подумать над самым подходящим и коротким относительным путем
  • Нажать во вкладке Element сочетание клавиш Ctrl+F (должна появиться строка «Find by string, selector, or XPath»
  • Ввести в эту строку относительный путь

Результатом такой проверки будет подсвеченный желтым цветом в части DOM элемент, путь для которого вы составили.


Комбинация абсолютных и относительных выражений

На практике часто используются комбинированные выражения, которые сочетают точность абсолютных путей и гибкость относительных. Например:

//p/input

Этот путь выбирает все элементы <input> внутри узла <p>, независимо от уровня вложенности.

Особенности выбора типа выражения

  1. Простые задачи:
    Для небольших и простых документов можно использовать абсолютные пути. Например, если вы тестируете небольшой HTML-формуляр, структура которого не меняется.
  2. Динамические страницы:
    Для страниц с динамическим контентом, где структура может изменяться (например, в веб-приложениях), относительные пути более предпочтительны.
  3. Оптимизация:
    Если документ содержит тысячи элементов, выбирайте путь, который минимизирует количество узлов, через которые нужно пройти.

Рекомендации по работе с типами выражений

  1. Используйте абсолютные пути для стабильных страниц:
    Например, административные панели, где структура страницы практически не меняется.
  2. Применяйте относительные пути для сложных структур:
    Особенно если нужно выбирать элементы по их атрибутам или текстовому содержимому.
  3. Избегайте избыточности:
    Если нужный элемент можно найти с помощью короткого относительного пути, нет необходимости строить длинный абсолютный.
  4. Комбинируйте подходы:
    Например, начните с абсолютного пути до ключевого узла, а затем используйте относительный путь для уточнения.

Примеры из практики

Поиск элемента с точным местоположением:

/html/body/div[1]/p[2]

Этот путь указывает на второй <p> в первом <div>.

Поиск элемента независимо от его местоположения:

//p[text()=’Пример текста’]

Находит элемент <p> с заданным текстом, независимо от его местоположения.

Сочетание методов:

//p[@class=’important’]/input

Находит все элементы <input>

Абсолютные пути обеспечивают точность, но чувствительны к изменениям структуры, тогда как относительные пути более гибкие, но требуют осторожности при применении. Выбор подходящего типа выражения зависит от задачи, структуры документа и требований к производительности.

Практическое задание Проговорите еще раз все плюсы и минусы относительного и абсолютного пути. Попробуйте самостоятельно отыскать абсолютный путь разных элементов страницы.

Базовые синтаксические элементы XPath

Чтобы эффективно использовать XPath, важно понимать его основные синтаксические элементы. Эти базовые составляющие служат строительными блоками для создания запросов, с помощью которых можно взаимодействовать с элементами и данными документа.

Что представляет собой структура документа?

HTML и XML-документы представляют собой деревья, где каждый элемент — это узел. Узлы могут быть:

  • Элементами (например, <div>, <p>).
  • Текстом (содержимое элемента).
  • Атрибутами (дополнительные параметры элементов, такие как id, class).
  • Комментарии или пространства имён, которые тоже считаются узлами.

XPath позволяет обращаться к этим узлам с помощью своей синтаксической системы.

Основные узлы в XPath

  1. Корневой узел Корневой узел (/) — это отправная точка документа. Он обозначает начало структуры. Например: / Этот символ используется для абсолютных путей, чтобы указать корень документа, от которого начинается поиск.
  2. Текущий узел Текущий узел (.) представляет собой элемент, от которого начинается запрос. Это полезно для относительных путей и уточнения поиска.
  3. Родительский узел Родительский узел (..) используется для перехода к родительскому элементу текущего узла. Например, если вы находитесь внутри <p>, то .. вернёт <div>, который является его родителем.
  4. Дочерние узлы Узлы, которые находятся внутри текущего узла, называются дочерними. XPath позволяет легко обращаться к ним. Например, выражение /html/body/div выберет узлы <div>, которые являются дочерними для <body>.
  5. Все узлы Символ * используется для выбора всех узлов текущего уровня. Это полезно, если вы хотите выбрать все элементы, не уточняя их теги. Например: /html/body/* Здесь будут выбраны все элементы, находящиеся внутри <body>.

Работа с атрибутами

Атрибуты в HTML и XML добавляют дополнительную информацию к элементам. XPath предоставляет синтаксис для выбора узлов атрибутов. Чтобы выбрать атрибут, используется символ @.

Выбор конкретного атрибута Вы можете выбрать атрибут узла, указав его имя: //div/@id

Это выражение выберет значение атрибута id всех элементов <div>.

Фильтрация по атрибутам Выражение в квадратных скобках позволяет уточнить поиск: //div[@class=’header’]

Здесь выбираются все <div> с атрибутом class, значение которого равно header.

Поиск узлов с любыми атрибутами Использование @* выбирает все атрибуты элемента: //div[@*]

Это выражение выберет все <div>, у которых есть хотя бы один атрибут.

Работа с текстом узлов

Текстовые узлы — это содержимое между открывающим и закрывающим тегами. XPath позволяет работать с ними с помощью функции text().

Выбор текста узла Если нужно выбрать только текстовое содержимое элемента: //h1/text()

Это выберет текст внутри всех элементов <h1>.

Поиск элементов по тексту Вы можете фильтровать элементы, основываясь на их содержимом: //p[text()=’Пример текста’]

Здесь выбираются только те <p>, текст которых равен «Пример текста».

Исключение текстовых узлов Если вы хотите работать только с элементами и игнорировать текст, просто не используйте text().

Индексация узлов

XPath поддерживает индексацию, позволяя выбирать узлы по их порядковому номеру.

Первый узел Чтобы выбрать первый элемент определённого типа: //ul/li[1]

Это выражение выберет первый элемент списка <li> в каждом <ul>.

Последний узел Для выбора последнего элемента используется функция last(): //ul/li[last()]

Она находит последний <li> в каждом списке <ul>.

Выбор диапазона узлов Вы можете выбрать несколько узлов сразу: //ul/li[position() <= 3]

Здесь будут выбраны первые три <li>.

Использование осей в XPath

Оси позволяют определять направления поиска относительно текущего узла.

Дочерние узлы (child) По умолчанию XPath выбирает дочерние узлы, но можно явно указать это: child::div

Это выберет все <div>, которые являются дочерними для текущего узла.

Родительский узел (parent) Возвращает родительский узел текущего элемента: parent::body

Это выберет <body>, если текущий узел находится внутри <body>.

Соседние узлы (following-sibling и preceding-sibling) Эти оси выбирают соседей текущего узла: //h2/following-sibling::p

Выберет все <p>, которые находятся на одном уровне с <h2> и следуют за ним.

Все предки (ancestor) Выбирает всех родителей текущего узла, включая <html>: ancestor::div

Найдёт всех предков типа <div>.

Все потомки (descendant) Находит всех потомков текущего узла, независимо от их уровня вложенности: descendant::span

Это выберет все <span>, которые находятся внутри текущего узла.

Практическое применение

Знание базовых синтаксических элементов XPath открывает широкие возможности для работы с документами. Например:

  • Выбор всех ссылок на странице: //a/@href
  • Поиск всех параграфов внутри секции: //section//p
  • Выбор только тех элементов, которые содержат подзаголовки: //div[h2]

Основные функции XPath

Функции в XPath расширяют базовые возможности запросов. Например, вместо того чтобы просто находить элемент, можно определить его точное местоположение, проверить его содержимое или выбрать элементы, которые соответствуют определённым условиям.

Основные функции XPath

  1. Работа с текстом
    • text() — одна из самых часто используемых функций, которая позволяет получить текстовое содержимое узла.
      Например: //p[text()=’Пример текста’]

Этот запрос найдёт все элементы <p>, текстовое содержимое которых равно «Пример текста».

  • contains() — эта функция проверяет, содержит ли текст определённое значение. Например://div[contains(text(), ‘Добро пожаловать’)]

Она найдёт все элементы <div>, текст которых содержит фразу «Добро пожаловать».

  • starts-with() — функция, которая проверяет, начинается ли текст с указанного значения. Например: //h1[starts-with(text(), ‘Заголовок’)]

Это выражение выберет все заголовки <h1>, которые начинаются с слова «Заголовок».

  • normalize-space() — удаляет начальные и конечные пробелы, а также заменяет последовательности пробелов внутри текста на один пробел. Например: //p[normalize-space(text())=’Текст без пробелов’]
  1. Работа с атрибутами
    • @* — позволяет выбрать все атрибуты узла. Например: //img[@*]

Этот запрос найдёт все <img> с любыми атрибутами.

  • @attribute-name — позволяет выбрать конкретный атрибут. Например: //a[@href]

Здесь выбираются все ссылки <a>, у которых есть атрибут href.

  • contains() также работает с атрибутами. Например: //button[contains(@class, ‘primary’)]

Это выражение выберет все кнопки <button>, у которых в атрибуте class содержится слово «primary».

  1. Работа с числами
    • position() — возвращает позицию узла относительно других узлов одного уровня. Например: //li[position()=2]

Этот запрос выберет второй элемент списка <li>.

  • last() — позволяет выбрать последний элемент. Например: //tr[last()]

Это выражение выберет последнюю строку <tr> в таблице.

  • Арифметические операции (+, -, *, div) используются для вычислений. Например: //price[. > 50]

Это выберет элементы <price>, значения которых больше 50.

  1. Условные выражения XPath поддерживает условные выражения для фильтрации узлов:
    • not() — проверяет, отсутствует ли указанное условие. Например: //input[not(@type=’hidden’)]

Здесь выбираются все элементы <input>, у которых атрибут type не равен «hidden».

  • and, or — используются для составления сложных условий. Например: //div[@class=’header’ and contains(text(), ‘Добро’)]

Это выражение выберет элементы <div>, у которых класс равен «header» и текст содержит слово «Добро».

  1. Работа с подвыражениями
    • count() — возвращает количество выбранных узлов. Например: count(//p)

Это вернёт количество элементов <p> в документе.

  • name() — возвращает имя узла. Например: //*[name()=’svg’]

Этот запрос найдёт все элементы <svg>.

  1. Работа с подстроками XPath предоставляет функции для работы с подстроками:
    • substring(string, start, length) — возвращает часть строки, начиная с определённой позиции. Например: substring(‘Пример строки’, 8, 6)

Вернёт «строки».

  • substring-after(string, substring) — возвращает часть строки после указанного подстрочечного значения. Например: substring-after(‘user@example.com’, ‘@’)

Вернёт «example.com».

  • substring-before(string, substring) — возвращает часть строки до указанного подстрочечного значения. Например: substring-before(‘user@example.com’, ‘@’)

Вернёт «user».

  1. Логические операции XPath поддерживает логические операции, которые используются для фильтрации:
    • true() и false() — возвращают логическое значение. Например: //input[@checked=true()]

Выбирает все элементы <input>, которые отмечены как checked.

  1. Группировка и объединение XPath позволяет объединять несколько выражений с помощью |. Например: //div | //span

Этот запрос найдёт все элементы <div> и <span> в документе.

Примеры реального применения

  1. Выбор кнопки «Отправить», содержащей слово «Отправить» в тексте:
  2. //button[contains(text(), ‘Отправить’)]
  3. Поиск всех изображений с альтернативным текстом:
  4. //img[@alt]
  5. Нахождение элементов списка с нечётной позицией:
  6. //li[position() mod 2 = 1]
  7. Подсчёт количества строк в таблице:
  8. count(//table/tr)

Советы по работе с функциями XPath

  1. Используйте функции для сокращения и уточнения запросов. Например, вместо того чтобы вручную перечислять все условия, используйте contains() или starts-with().
  2. Проверяйте работоспособность запросов в браузерных DevTools или специальных редакторах XPath.
  3. При сложных запросах комбинируйте функции. Например:
  4. //div[contains(@class, ‘important’) and not(@hidden)]
  5. Следите за производительностью. Сложные выражения с использованием функций могут замедлить обработку больших документов.
